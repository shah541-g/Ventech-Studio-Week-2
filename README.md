# Week 2 Directory Walkthrough

This Week 2 workspace is my working record of how I learned HTML, CSS, layout systems, and a little JavaScript by building small projects, challenge solutions, and theory notes. I have treated it like a growth log: first I studied the concepts, then I practiced them, then I applied them in real layouts.

## Content

This folder is my study space. It contains standalone HTML notes that I used to revise core CSS topics.

- `Box Model.html` — I reviewed spacing, sizing, and how element boxes behave.
- `Display.html` and `Display Content.html` — I studied how layout behavior changes with different display values.
- `Flexbox.html` — I built a flexbox recall page to understand axes, alignment, wrapping, and item growth.
- `Grid.html` — I wrote a detailed grid recall page to learn tracks, lines, areas, spanning, and responsive grids.
- `Image.html` — I revisited image handling and responsive presentation.
- `Media Queries.html` — I practiced breakpoint-driven responsiveness.
- `Overflow.html` and `Position.html` — I learned how content clipping and positioning affect layout.
- `Pseudo Classes.html` — I reinforced interactive selector behavior.
- `Scrollbar Styling.html` — I explored browser scrollbar customization.
- `Shadows.html` — I practiced depth and elevation styling.
- `Units.html` — I reviewed px, rem, %, fr, and other sizing units.
- `Additional/` — This subfolder stores PDF notes and external references I used to support my learning, including CSS notes from videos, roadmaps, and web.dev.

### What I learned here

I did not just memorize properties; I started understanding how CSS concepts connect. The content folder helped me move from isolated facts to a mental model of layout.

## Worked Examples

This folder contains guided builds that I used to practice turning ideas into working interfaces.

- `Navbar Task/` — I built a navigation layout with `index.html`, `index.css`, and an image asset for the navbar.
- `Card Grid Task/` — I created a service-style card grid with a sidebar toggle, multiple visual assets, and a responsive layout structure.

### What I learned here

I learned how to translate a reference design into semantic HTML and structured CSS. These examples helped me become more comfortable with reusable sections, navigation patterns, and visual hierarchy.

## Practice Tasks

This folder contains my hands-on exercises where I tried to build layouts without depending on a finished scaffold.

- `Redesigning a Mockup/` — I rebuilt a mockup-driven page and compared my output against the original mockup reference.
- `3-Pricing Cards/` — I practiced building a clean pricing-card layout with consistent spacing and alignment.
- `Profile Page Task/` — I built a profile page with HTML, CSS, JavaScript, supporting images, and a VS Code settings file for the task.

### What I learned here

I learned how to work more independently. Instead of only following instructions, I started making layout decisions, testing responsive behavior, and using JavaScript only where interaction was needed.

## Weeks Deliverable

This folder holds the more complete deliverables for the week.

- `Profile Page Task/` — I extended the profile-page idea into a fuller multi-page experience with `index.html`, `about.html`, `contact.html`, CSS, JavaScript, and supporting images.
- `Grid Based Dashboard/` — I built a dashboard interface with a responsive header, sidebar navigation, stat cards, SVG assets, design references, and a small script for toggling the sidebar.

### What I learned here

I learned how to assemble larger page structures and keep multiple parts of the interface working together. This is where I improved most in responsive layout planning, component grouping, and interactive state handling.

## My Learning Journey

Across this Week 2 workspace, I have developed a stronger grip on:

- semantic HTML structure
- CSS box model and spacing discipline
- flexbox and grid layout thinking
- responsive design choices

## How I approached the week

1. I studied the topic in `Content/`.
2. I practiced it in `Worked Examples/`.
3. I tried it on my own in `Practice Tasks/`.
4. I packaged the stronger outcomes in `Weeks Deliverable/`.
5. I validated my understanding again through `Design Challenges/`.

That loop helped me move from theory to application, and from application to reflection.
