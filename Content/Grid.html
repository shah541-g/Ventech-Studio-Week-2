🔲 CSS Grid — Active Recall Notes

Video: Grid basics, positioning, implicit grid, fr unit, template areas, alignment, responsive trick

1. Grid Ki Basic Terminology

Grid two-dimensional hai — items horizontally, vertically, ya dono ek sath position ho sakte hain, yahan tak ke stacked (layered) bhi.

  • Lines — har number ek line represent karta hai. Row lines (horizontal) aur column lines (vertical)
  • Cell — grid ka har chhota square
  • Track — rows aur columns (poori row ya poori column ek "track" hai)

2. Grid Banana

.container {
  display: grid;
  grid-template-columns: 100px 100px 100px;
  grid-template-rows: 100px 100px;
}

grid-template-columns / grid-template-rows — har value ek track define karti hai. 3 values dene se 3 columns ban jaate hain.

⚠️ Sirf display: grid lagane se kuch visible change nahi hota — rows/columns explicitly define karne padte hain.

3. Item Position Karna — Line-based

.item-1 {
  grid-row-start: 1;
  grid-row-end: 3;
  grid-column-start: 1;
  grid-column-end: 5;
}

Line numbers use hoti hain — item line 1 se shuru hokar line 3 tak rows mein, aur line 1 se line 5 tak columns mein span karega.

⚠️ Agar baaki items ki position explicitly define na ki jaaye, to wo automatically push ho jaate hain.

4. Shorthand: grid-row / grid-column

.item-2 {
  grid-row: 1 / 3;
  grid-column: 5 / 7;
}

Format: start-line / end-line. 4 alag properties likhne se behtar shorthand hai.

5. span Keyword

.item-2 {
  grid-row: span 2;
  grid-column: span 2;
}

span ka matlab: "jahan bhi ho, wahan se itni cells span kar jao" — explicit line numbers ki zaroorat nahi, easy to read.

⚠️ Downside: span explicit start/end position define nahi karta — agar koi nearby item ka size change ho jaaye, to ye item bhi push ho sakta hai (position "anchored" nahi hoti).

6. grid-area (4-value Shorthand)

Sab se fast property — 4 values ek sath (row-start, column-start, row-end, column-end):

.item-3 {
  grid-area: 3 / 1 / 7 / 7;
}

Isse 6 properties ka kaam ek line mein ho jaata hai.

Negative line numbers bhi use ho sakti hain — -1 grid ki aakhri line ko represent karta hai:

.item-3 {
  grid-area: 3 / 1 / -1 / -1; /* end tak span */
}

7. Layering (Stacking Items)

Grid se pehle layering ke liye position: absolute + top/right/bottom/left ka jhanjhat karna padta tha. Grid mein bas item ko us position par rakh do, agar wo kisi doosre item se overlap kare to automatically layering ban jaati hai.

Layer order control karne ke liye normal z-index use hota hai:

.item-2 { z-index: 1; }

8. Implicit Grid

Agar explicitly defined grid mein jagah khatam ho jaaye aur naya item add ho, to grid automatically ek naya row/column bana leta hai — isse implicit grid kehte hain.

⚠️ Implicit grid ki rows/columns grid-template-rows/grid-template-columns ki values ko inherit nahi karti — isliye naya item size mein alag/chhota dikhta hai. 
.container {
  grid-auto-rows: 100px;       /* implicit rows ka size */
  grid-auto-flow: column;       /* default naya row ki jagah column banega */
  grid-auto-columns: 100px;    /* implicit columns ka size */
}

9. Units: fr, minmax, repeat

fr (Fractional Unit)

grid-template-columns: 1fr 1fr 1fr;

Available space ka proportional (fractional) hissa represent karta hai. Sab 1fr ho to teeno columns equal space lenge. Value badhane se (jaise 2fr) wo column zyada space le lega.

fr ko doosre units (px, %, rem) ke saath mix bhi kar sakte hain: 100px 1fr 1fr.

minmax()

grid-template-columns: 1fr minmax(100px, 3fr) 1fr;

Ek track ke liye minimum aur maximum size define karta hai — item us range ke andar hi resize hoga, us se chhota nahi hoga.

repeat()

grid-template-rows: repeat(2, 100px);

Pehla argument = kitni baar repeat karna hai, dusra argument = kya value repeat karni hai. Repetition avoid karta hai.

10. gap

.container {
  gap: 20px 10px; /* pehli value = row gap, dusri = column gap */
}

Ek value dene se rows aur columns dono par equally apply hoti hai.

11. grid-template-areas

Line numbers track kiye bagair items position karne ka tareeka — named sections use karta hai.

.container {
  grid-template-rows: auto 1fr auto;
  grid-template-columns: 1fr 1fr;
  grid-template-areas:
    'header header'
    'main aside'
    'footer footer';
}

.item-1 { grid-area: header; }
.item-2 { grid-area: main; }
.item-3 { grid-area: aside; }
.item-4 { grid-area: footer; }
  • Har set of quotes ('...') ek row represent karta hai
  • Har quote ke andar values ki tadaad columns ki tadaad ke barabar honi chahiye
  • Items ko sirf naam se assign karo: grid-area: header;
⚠️ Ye tareeka simple aur readable hai, lekin isme layering (items ko overlap) karna line-numbers wale tareeke se zyada mushkil hai.

12. justify-items / align-items (Container)

Default value: stretch (isliye items apni cell ko poora fill karte hain).

  • justify-items — row axis (horizontal) par items align karta hai
  • align-items — column axis (vertical) par items align karta hai

Values: stretch (default), start, end, center, baseline.

13. justify-self / align-self (Item)

Individual item ke liye container ke justify-items/align-items ko override karta hai:

.item-1 {
  justify-self: center;
  align-self: stretch;
}

14. justify-content / align-content (Poori Grid)

Ye poore grid ko container ke andar align karte hain (jab grid container se chhota ho) — justify-items/align-items se confuse na karo, wo individual items ko cell ke andar align karte hain.

  • justify-content — grid ko row axis par align
  • align-content — grid ko column axis par align

Values: start, center, baseline, space-between, space-around, space-evenly.

15. Responsive Grid — Zero Media Queries!

.container {
  grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
}

repeat(4, ...) ki jagah auto-fit keyword use karo. Jab items apni minimum size (minmax se defined) tak pohanch jaayein, to auto-fit automatically unhe agli row mein wrap kar deta hai — bina kisi media query ke, purely responsive layout.

📝 Active Recall

Question par click karo — answer reveal hoga. Khud pehle try karo, phir check karo.

CSS Grid two-dimensional hone ka kya matlab hai?
Items horizontally, vertically, aur dono directions mein simultaneously position kiye ja sakte hain — yahan tak ke items stack (layer) bhi kiye ja sakte hain.
Grid ki 3 basic terminology kya hain?
Lines (row lines aur column lines), Cells (har chhota square), aur Tracks (poori row ya poori column).
grid-template-columns mein har value kya represent karti hai?
Ek track (yani ek column) — jitni values dogay, utne columns ban jaayenge us size ke.
grid-row aur grid-column shorthand ka format kya hai?
start-line / end-line — dono values ek slash se separate hoti hain, jaise grid-row: 1 / 3;
span keyword ka main downside kya hai?
Ye explicit start aur end position define nahi karta, sirf "yahan se itni cells span karo" batata hai — isliye agar nearby item ka size change ho, to ye item bhi push ho sakta hai (position anchored nahi hoti).
grid-area shorthand mein 4 values kis order mein hoti hain?
row-start / column-start / row-end / column-end.
Grid mein negative line number (jaise -1) kya represent karta hai?
Grid ki aakhri line — isse aap explicit last number pata kiye bagair grid ke end tak span kar sakte ho.
CSS Grid mein items ko layer (stack) karna position: absolute ke comparison mein kyun aasan hai?
Bas item ko us position par rakh do jahan wo doosre item ke sath overlap ho jaaye — automatically layering create ho jaati hai, top/right/bottom/left manually calculate nahi karne padte. Layer order ke liye normal z-index use hoti hai.
Implicit grid kab create hoti hai?
Jab explicitly defined grid mein saari jagah occupy ho chuki ho aur ek naya item add kiya jaaye — grid automatically ek naya row (ya column) bana deta hai us item ko accommodate karne ke liye.
Implicit grid mein naya item chhota kyun dikhta hai?
Kyunke implicit grid ke rows/columns, grid-template-rows aur grid-template-columns ki defined values ko inherit nahi karte — jab tak grid-auto-rows/grid-auto-columns explicitly na diya jaaye.
grid-auto-flow: column set karne se kya change hota hai?
Default behavior (naya implicit row banana) ki jagah, ab naya implicit content column ki tarah add hoga.
fr unit kya represent karta hai?
Available space ka ek fractional (proportional) hissa — jaise 1fr 1fr 1fr se teeno tracks available space ko barabar baant lete hain.
minmax(100px, 3fr) kya karta hai?
Us track ke liye minimum size 100px aur maximum size 3fr set karta hai — track us range se chhota nahi ho sakta.
repeat(2, 100px) ka matlab kya hai?
100px value ko 2 baar repeat karo — yani do tracks banao, dono 100px ki. Pehla argument repetition count hai, dusra argument repeat hone wali value hai.
gap: 20px 10px; mein kaunsi value row gap hai aur kaunsi column gap?
Pehli value (20px) row gap hai, dusri value (10px) column gap hai.
grid-template-areas mein har set of quotes kya represent karta hai, aur andar ki values kya represent karti hain?
Har set of quotes (' ') ek row represent karta hai. Uske andar likhi har value ek column ko us named section se assign karti hai.
grid-template-areas approach ka main downside kya hai?
Isse items ko layer (overlap) karna, line-number based approach ke comparison mein zyada mushkil hota hai.
justify-items aur align-items (container par) mein kya farq hai?
justify-items row axis (horizontal) par items align karta hai apni cell ke andar. align-items column axis (vertical) par align karta hai. Dono ki default value stretch hoti hai.
justify-self / align-self kis cheez ko override karte hain?
Container ke justify-items / align-items ko — lekin sirf ek individual item ke liye, baaki items par container ki default value hi apply hoti rehti hai.
justify-content / align-content, justify-items / align-items se kaise different hain?
justify-content/align-content poore grid ko container ke andar align karte hain (jab grid container se chhota ho). justify-items/align-items individual items ko unki apni cell ke andar align karte hain.
Bina media query ke responsive grid banane ka tareeka kya hai?
grid-template-columns: repeat(auto-fit, minmax(100px, 1fr)); — repeat count ki jagah auto-fit keyword use karo, isse items apni minimum size tak pohanchte hi automatically agli row mein wrap ho jaate hain.
