🧩 CSS Flexbox — Active Recall Notes

Video: Flexbox fundamentals — axes, alignment, wrapping, growth/shrink, order

0. Container vs Item Properties

Container par: display: flex flex-direction justify-content align-items flex-wrap align-content gap

Direct Children (items) par: flex-grow flex-shrink flex-basis flex (shorthand) align-self order

1. Do Invisible Axes

display: flex lagate hi container mein 2 invisible axes ban jaati hain:

  • Main axis — items ismein primarily position hote hain
  • Cross axis — main axis ke perpendicular

Default: main axis horizontal hoti hai (kyunke flex-direction: row default hai).

2. flex-direction (Main Axis Ka Direction)

ValueMain AxisCross Axis
row (default)Horizontal (left → right)Vertical (top → bottom)
columnVerticalHorizontal
⚠️ Yehi Flexbox samajhne ki key hai: justify-content hamesha main axis par kaam karta hai — chahe wo horizontal ho ya vertical (flex-direction se decide hota hai).

3. justify-content (Main Axis Alignment)

ValueKaam
flex-start (default)Items line ke start par
flex-endItems line ke end par push
centerItems beech mein center
space-betweenPehle aur last item ke darmiyan evenly distribute (edges par space nahi)
space-aroundspace-between jaisa, lekin edges par bhi kuch spacing
space-evenlySab items ke darmiyan (aur edges par bhi) bilkul equal spacing

4. align-items (Cross Axis Alignment)

ValueKaam
flex-startItems cross axis ke start par
flex-endItems cross axis ke end par
centerItems cross axis ke beech mein
baselineItems is tarah align hote hain ke sab ke text ki baseline (chahe font-size different ho) ek line mein aa jaaye
⚠️ Yaad rakho: justify-content = main axis, align-items = cross axis. Ye distinction hi Flexbox samajhne ka core hai.

5. flex-wrap

Default: nowrap — items zabardasti ek hi line mein fit hone ki koshish karte hain, jagah kam pade to items crush (squeeze) hone lagte hain.

.container {
  flex-wrap: wrap;
}

Isse items ab ek line mein fit hone ki zid nahi karte — jab space khatam ho jaaye to naye line mein wrap ho jaate hain.

6. align-content (Sirf Wrap Ke Sath)

align-content tabhi kaam karta hai jab flex-wrap: wrap ho aur items multiple lines mein wrap ho rahe hon. Ye cross axis par saare lines ko together align karta hai (align-items se confuse na karo, wo ek line ke andar items align karta hai).

Possible values justify-content jaisi hi: flex-start, flex-end, center, space-between, space-around, space-evenly.

7. gap

Items ke darmiyan direct spacing dene ke liye — container par lagayi jaati hai:

.container {
  gap: 1rem;
}

8. flex-grow (Item Property)

Unitless value jo proportion ki tarah kaam karti hai — agar container mein extra space bachi ho, to ye define karta hai item kitna grow kare.

.item-3 { flex-grow: 1; }
  • Agar sirf ek item par flex-grow: 1 ho, to saari remaining space wahi item le lega
  • Agar sab items par flex-grow: 1 ho, to remaining space sab mein equally distribute hogi

9. flex-shrink (Item Property)

Unitless value — define karta hai item kitni tezi se shrink ho jab space kam pade (baaki items ke comparison mein).

.item-1 { flex-shrink: 5; }  /* baaki se zyada tezi se shrink hoga */
.item-1 { flex-shrink: 0; }  /* bilkul shrink nahi hoga */

10. flex-basis (Item Property)

Item ka size define karta hai before remaining space distribute ho — agar item par pehle se width set hai, to flex-basis usko overwrite kar deta hai.

.item {
  width: 150px;
  flex-basis: 300px; /* ab actual size 300px ho jaayega */
}

flex-basis: 0 item ko max tak shrink kar deta hai (starting point 0 maan liya jaata hai).

11. flex (Shorthand)

flex = flex-grow + flex-shrink + flex-basis ka combined shorthand. Doosri aur teesri value optional hain.

.item { flex: 1; } /* baaki 2 values automatically/intelligently set ho jaati hain */

Practically individual grow/shrink/basis se zyada ye shorthand hi use hota hai.

12. align-self (Item Property)

Container ke align-items ko ek specific individual item ke liye override karta hai.

.container { align-items: flex-start; }
.item-1 { align-self: center; } /* sirf ye item center hoga */

Possible values: flex-start, flex-end, center, baseline.

13. order (Item Property)

Default: har item ka order: 0 hota hai, isliye wo HTML order mein hi appear karte hain. Isko change karke visual order badla ja sakta hai (HTML change kiye bagair).

.item-3 { order: -1; } /* teesra item ab sab se pehle dikhega */
.item-1 { order: 1; }  /* pehla item ab sab se aakhir mein dikhega */
⚠️ Best practice: order ka use avoid karo jab tak zaroori na ho — ye HTML ki semantics aur accessibility ko disturb karta hai (screen readers HTML order follow karte hain, visual order nahi).

📝 Active Recall

Question par click karo — answer reveal hoga. Khud pehle try karo, phir check karo.

display: flex lagane se container mein kya create hota hai?
Do invisible axes — main axis aur cross axis. Items in axes ke around position hote hain.
Default main axis kis direction mein hoti hai aur kyun?
Horizontal — kyunke flex-direction ki default value row hoti hai.
flex-direction: column set karne se main axis kya ban jaati hai?
Vertical — aur cross axis horizontal ban jaati hai.
justify-content kis axis par kaam karta hai?
Main axis par (chahe wo row ho ya column, flex-direction jo bhi ho).
align-items kis axis par kaam karta hai?
Cross axis par (main axis ke perpendicular).
space-between aur space-around mein kya farq hai?
space-between: pehle aur last item ke darmiyan evenly space distribute hoti hai, edges par koi extra space nahi. space-around: waise hi lekin edges par bhi thori spacing hoti hai.
align-items: baseline kya karta hai?
Items ko is tarah align karta hai ke unke text ki baseline ek line mein aa jaaye, chahe alag-alag items ka font-size different kyun na ho.
Default flex-wrap value kya hoti hai, aur uska result kya hota hai jab items ke liye jagah kam pad jaaye?
nowrap — items ek hi line mein fit hone ki koshish karte hain, aur agar jagah kam ho to wo aapas mein crush (squeeze) hone lagte hain.
align-content property kab activate/use hoti hai?
Sirf tab jab flex-wrap: wrap set ho aur items actually multiple lines mein wrap ho rahe hon — ye saari lines ko cross axis par together align karta hai.
align-content aur align-items mein farq kya hai?
align-items ek hi line ke andar items ko cross axis par align karta hai. align-content poori (wrapped) lines ko cross axis par ek group ki tarah align karta hai — sirf wrap hone par kaam karta hai.
flex-grow: 1 sirf ek item par lagane se kya hota hai (agar container mein extra space ho)?
Wo item saari remaining/extra space le leta hai aur grow ho jaata hai.
Agar sab items par flex-grow: 1 set ho, to remaining space kaise distribute hoti hai?
Sab items mein equally (proportionally) distribute hoti hai.
flex-shrink: 0 lagane se item ka behavior kya hoga?
Wo item bilkul shrink nahi hoga, chahe container mein jagah kam ho jaaye.
flex-basis kya define karta hai aur ye existing width property ko kaise affect karta hai?
Item ka size (remaining space distribute hone se pehle) define karta hai — agar item par pehle se width set hai, to flex-basis usko overwrite kar deta hai.
flex shorthand mein kaunsi 3 properties combine hoti hain?
flex-grow, flex-shrink, aur flex-basis — jismein dusri aur teesri value optional hoti hain aur automatically set ho jaati hain.
align-self kis property ko override karta hai, aur kis level par?
Container ke align-items ko override karta hai, lekin sirf ek individual item ke level par — baaki items par container ka align-items normal tarah apply hota rehta hai.
order property ki default value kya hoti hai, aur is se kya implication hoti hai?
Default order: 0 hota hai sab items ka — isliye wo HTML mein jis order mein likhe hon, usi order mein visually appear karte hain.
order property ka use kyun avoid karna chahiye jab tak zaroori na ho?
Kyunke ye HTML document ki semantics aur accessibility ko disturb karta hai — screen readers aur assistive tech HTML ka actual order follow karte hain, visual (CSS) order nahi.
