🧱 CSS Display Properties — Active Recall Notes

Video: Block, Inline, Inline-Block, None/Opacity/Visibility, Flex, Grid

1. Display: Block

Default behavior — element poori available width le leta hai aur har naya element next line par drop hota hai.

  • Width, height, padding, margin — sab kuch normally apply hota hai
  • Examples: div, h1–h6, p, ul, li

2. Display: Inline

Default for: a, span, img, strong, small.

⚠️ Inline elements sirf utni space lete hain jitni text render karne ke liye zaroori ho. Ye ek dusre ke saath (side by side) aate hain, next line par nahi jaate.

Limitations:

  • width aur height set karna kaam nahi karta
  • padding aur margin — horizontal spacing apply hoti hai, lekin vertical spacing baaki elements ke liye ignore ho jaati hai (visually padding dikhti hai par layout shift nahi hoti)

Use-case: paragraph ke andar kisi specific word/phrase ko style karna, jaise <span> se bold/red text.

3. Display: Inline-Block

Best of both worlds — elements side by side aate hain (jaise inline), lekin width, height, padding, aur margin (vertical bhi) sab kaam karte hain (jaise block).

Default example: button element inline-block hota hai.

li {
  display: inline-block;
  padding: 1rem;
}

Isse list items ek line mein aa jaate hain aur unpe padding bhi normally kaam karta hai.

4. Hide Karne Ke 3 Tareeke

opacity: 0 ka concept white-text-on-white-background jaisa hai — dikhta nahi lekin wahin maujood hai.

5. Display: Flex

Sabse popular aur simple layout method — one-directional (ek waqt mein sirf row ya column).

ul {
  display: flex;
  flex-direction: row; /* ya column */
  justify-content: center;
  align-items: center;
  gap: 2rem;
}
  • flex-direction: row (default) — items horizontally ek line mein
  • flex-direction: column — items vertically stack hote hain
  • justify-content — main axis par alignment (center, flex-start, flex-end, space-around/between/evenly)
  • align-items — cross axis par alignment
⚠️ Important: jab flex-direction: row → column switch karte ho, to justify-content aur align-items ka role reverse ho jaata hai — jo pehle horizontal control karta tha wo ab vertical control karega.
  • gap — items ke darmiyan spacing (padding ki zaroorat nahi)
  • align-self — individual child element ki position container ke default alignment se alag override karne ke liye

6. Display: Grid

Flexbox jaisa hi hai, lekin two-dimensional — columns aur rows dono ek saath control hote hain. Isliye flexbox se zyada powerful lekin zyada complex bhi.

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 1fr 1fr 1fr;
}
  • grid-template-columns: 1fr 1fr 1fr = repeat(3, 1fr) — teen equal-width columns
  • Agar rows define na karein, extra elements ke liye grid automatically naye rows bana deta hai

Element Position Karna

Grid lines 1 se count honi shuru hoti hain (chahe row ho ya column):

grid-row: 2 / 3;     /* dusri row mein */
grid-column: 5 / 6;  /* paanchvi column mein */

Use-case: complex layouts (jaise photo gallery grids) jahan flexbox se bohot saare nested divs lagte — grid mein ek hi container se ho jaata hai.

7. Kam Use Hone Wale Displays

display: table, table-cell, list-item waghera bhi maujood hain jo element ko us tag ki tarah behave karwate hain, lekin practically bohot kam use hote hain.

📝 Active Recall

Question par click karo — answer reveal hoga. Khud pehle try karo, phir check karo.

Display block ka default behavior kya hota hai?
Poori available horizontal width le leta hai aur agla element automatically next line par chala jaata hai.
Inline elements par width/height set karne ki koshish karo to kya hota hai?
Kuch nahi hota — inline elements par explicit width aur height kaam nahi karte (jaise a, span, img, strong, small).
Inline element par padding lagayein to kya farq padta hai?
Padding visually dikhti hai (horizontal spacing bhi apply hoti hai), lekin vertical padding ki wajah se dusre elements apni position change nahi karte — wo is vertical space ko ignore kar dete hain.
Inline-block, inline aur block se kaise mix hota hai?
Elements side-by-side aate hain (inline ki tarah), lekin width, height, aur vertical padding/margin bhi kaam karte hain (block ki tarah). Button element by default inline-block hota hai.
display: none, opacity: 0, aur visibility: hidden mein kya farq hai?
display:none — element DOM se layout ke lihaz se hat jaata hai, space collapse ho jaata hai, animate nahi ho sakta. opacity:0 — space reserved rehti hai, element phir bhi selectable/clickable hota hai, animate ho sakta hai. visibility:hidden — space reserved rehti hai lekin element select nahi ho sakta.
display:none ko animate kyun nahi kiya ja sakta?
Kyunke ye ek on/off switch ki tarah kaam karta hai — sirf none se block (ya vice versa) jump karta hai, koi gradual in-between state nahi hoti (opacity ki tarah 0 se 1 tak).
Flexbox one-directional kyun kehlata hai?
Kyunke ek waqt mein flex-direction sirf row ya column mein se ek hoti hai — items ya to horizontally line up hote hain ya vertically, dono ek saath nahi.
flex-direction ko row se column mein change karne par kya reverse ho jaata hai?
justify-content aur align-items ka role reverse ho jaata hai — jo pehle horizontal alignment control karta tha (justify-content), column mein switch karne ke baad wo vertical control karne lagta hai, aur align-items horizontal control karta hai.
Flexbox mein items ke darmiyan spacing dene ka aasan tareeka kya hai (padding ke bagair)?
gap property — jaise gap: 2rem;, ye directly container par lagti hai.
Flex container ke andar ek individual child element ki position alag se override karne ke liye kya use hota hai?
align-self — is se container ke default align-items se hat kar us specific element ki apni alignment set ki ja sakti hai.
Grid, flexbox se kaise different hai?
Flexbox one-dimensional hota hai (row ya column), jabke Grid two-dimensional hai — rows aur columns dono ek saath define aur control kiye ja sakte hain. Isse zyada powerful hai lekin thora zyada complex bhi.
grid-template-columns: repeat(3, 1fr); ka matlab kya hai?
Teen equal-width columns banao, har column available space ka barabar hissa (1 fraction) le. Ye 1fr 1fr 1fr likhne jaisa hi hai.
Grid lines ka counting kahan se start hoti hai?
1 se — chahe row ho ya column, dono directions mein grid lines 1 se count honi shuru hoti hain.
Agar grid mein columns define hon lekin rows na ki gayi hon, aur elements zyada hon to kya hota hai?
Grid automatically extra rows create kar deta hai jitni zaroorat ho, taake baaki elements accommodate ho sakein.
