1. Media Queries Kya Hain?

Media queries developers ko device ya viewport ki characteristics ke basis par conditional styles apply karne dete hain — isi wajah se ek website mobile aur desktop dono par sahi dikhti hai.

Zaroori Meta Tag

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Ye tag ensure karta hai ke browser client ki actual screen width ko accurately detect kare — is ke bagair media queries sahi kaam nahi karti.

2. Media Rule Ka Basic Syntax

@media (max-width: 600px) {
  body {
    background-color: blue;
  }
}

Ye rule kehta hai: jab viewport width 600px ya usse kam ho, to body ka background blue ho jaaye.

3. max-width Property

max-width media query un screens ko target karti hai jo specified breakpoint se chhoti/barabar hon.

Use-case example: ek nav bar jo desktop ke liye design ki gayi hai, phone par sahi fit nahi hoti — max-width breakpoint use karke text, logo, aur links ki size adjust ki ja sakti hai taake nav bar phone par bhi theek dikhe.

Testing ke liye Chrome DevTools ka "toggle device bar" use kiya ja sakta hai — real-time mein different screen sizes preview karne ke liye.

4. Common Device Breakpoints

DeviceTypical Width Range
📱 Phones320px – 600px
💻 Tablets600px – 1024px
🖥️ Desktops/Laptops1024px – 1440px

Example (Flexbox cards):

  • Desktop: flex-direction: row — cards ek line mein
  • Tablet: cards ki height/width chhoti kar dena
  • Phone: flex-direction: column — cards vertically stack

5. Mobile-First vs Desktop-First

ApproachKaise Kaam Karta Hai
Mobile-First Default styles sabse chhoti screen ke liye likhte hain, phir min-width media queries se badi screens ke liye styles add karte hain
Desktop-First Default styles desktop ke liye, phir max-width se chhoti screens ke liye styles override karte hain
⚠️ min-width, max-width ka opposite hai — ye specified breakpoint se badi/barabar screens ko target karta hai.

Zyada developers mobile-first approach prefer karte hain, kyunke mobile traffic desktop se zyada hai:

Mobile: 60.74%   Desktop: 37.1%

(StatCounter Global Stats, June 2024)

6. Fluid Responsive Design Ke Key Concepts

Media Scaling (Images/Videos)

img, video {
  max-width: 100%;
  height: auto;
}

Isse media apne container se overflow nahi karti aur proportionately resize hoti hai.

Relative Units

Fixed units (px) ki jagah relative units — %, em, rem — use karna design ko zyada flexible banata hai.

Scalable Typography

Font-sizes, line-heights, aur spacing ko viewport ke hisaab se adjust karne ke liye:

  • calc() function
  • vw / vh units

Isse text har device par readable rehta hai.
