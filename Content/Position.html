📌 CSS position Property — Active Recall Notes

Video: static, relative, fixed, sticky, absolute + z-index

1. Overview — 5 Possible Values

  • static — default, koi effect nahi
  • relative
  • fixed
  • sticky
  • absolute

static ke ilawa baaki sab par top, bottom, left, right, aur z-index properties use ki ja sakti hain element ko exactly position karne ke liye.

2. position: relative

Element ko uski apni normal (static) position ke relative offset karta hai.

.box {
  position: relative;
  top: 30px;
  left: 30px;
}
  • top: 30px → element apni original position se 30px neeche push hoga
  • left: 30px → element apni original position se 30px right push hoga
  • Values jitni zyada, element utna hi door push hoga
  • bottom ya right bhi use ho sakte hain — par ye element ko screen se bahar push kar sakte hain (kyunke hamesha static position se "away" push hota hai)

3. position: fixed

Element ko window/viewport ke sath fix kar deta hai — scrollable page par ye user ke saath scroll karega (follow karega), kyunke ye ek naye stacking context / layer par chala jaata hai.

⚠️ Behavior change: fixed elements mein top, left, right, bottom ab element ki apni position ke relative nahi, balke poori window ke relative hote hain.
  • right: 0 → element right edge se 0px door (yani chipka hua)
  • top: 0 → element top edge se 0px door

Percentage Values + Centering Trick

Fixed positioning mein percentage values bhi use ho sakti hain. Center karne ke liye:

.box {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
  • Sirf top: 50%; left: 50%; lagane se element ka top-left corner center hota hai (element ka origin), poora element center nahi hota
  • Isko fix karne ke liye transform: translate(-50%, -50%) add karo — ye element ko uski apni width/height ke 50% jitna wapas push karta hai, taake actual center align ho jaaye

Ye wahi trick hai jo cookie-consent popups aksar center karne ke liye use karte hain.

4. position: sticky

Normal element ki tarah behave karta hai, lekin scrollable container mein ek specific point tak pohanchne par ye "chipak" (stick) jaata hai — jaise fixed element hota hai.

  • Sticking point ko top, left, right, bottom se define karte hain
  • Stick hone ke baad element bhi naye stacking context par chala jaata hai (fixed jaisa)
⚠️ Fixed aur sticky mein farq: fixed hamesha fixed rehta hai. Sticky sirf uss point tak pohanchne ke baad stick hota hai, us se pehle normal flow mein rehta hai.

5. position: absolute

Sab se important position value. Ek absolute-positioned element hamesha apne nearest ancestor ke relative position hota hai — jis ancestor ka koi bhi position value specified ho (static ke ilawa).

Parent-Child Setup

.parent {
  position: relative; /* reference point banata hai */
}
.child {
  position: absolute;
  top: 0;
  right: 0;
}
  • Parent par position: relative lagana best practice hai, kyunke iska khud par almost koi visual impact nahi hota (jab tak top/left/right/bottom na diya jaaye)
  • Child par top: 0; right: 0; → child parent ke top-right corner mein position ho jaata hai
⚠️ Agar parent par koi positioned value (relative/absolute/fixed) na ho, to absolute child body ke relative position hoga — jo aksar unwanted result deta hai.

Practical example: search bar ke andar ek button ko position karna — button par position: absolute aur search bar wrapper par position: relative.

6. Stacking Context aur z-index

fixed aur absolute dono elements ko ek naye stacking context mein daal dete hain — matlab ye baaki website ke elements ke upar (cover) aa sakte hain.

z-index se layers ka order control karte hain:

  • Negative values → element ko baaki elements ke peeche bhejte hain
  • Higher (positive) values → element ko baaki elements ke upar bhejte hain

Use-cases: navigation bar, sidebar, overlay, popup — inko baaki website se higher z-index dena chahiye taake ye hamesha upar dikhein.

position ki 5 possible values kaunsi hain?
static, relative, fixed, sticky, absolute.
position: relative mein top aur left kis cheez ke relative hote hain?
Element ki apni normal (static) position ke relative — yani jahan wo hota agar position static hoti.
position: relative ke sath top: 30px aur left: 30px dene se element kis taraf move hoga?
Apni original position se 30px neeche (down) aur 30px right (dayein) push hoga.
position: fixed lagane se scroll karne par element ka kya hota hai?
Element user ke saath follow karta hai — screen par fix rehta hai, scroll hone par apni jagah nahi chhorta, kyunke ye ek naye stacking context/layer par chala jaata hai.
position: fixed mein top/left/right/bottom kis cheez ke relative hote hain?
Poori window/viewport ke relative — element ki apni position ke relative nahi (jaise relative mein hota tha).
Fixed element ko screen ke bilkul center mein karne ke liye kya karna padta hai?
top: 50%; left: 50%; dene ke baad transform: translate(-50%, -50%); bhi lagana padta hai, warna sirf element ka top-left corner center hota hai, poora element nahi.
top: 50%; left: 50%; akele lagane se element center kyun nahi hota?
Kyunke ye sirf element ke top-left corner (origin) ko screen ke 50%/50% point par le jaata hai — element ka baaki hissa us point se aage right aur neeche extend hota hai, isliye visually center nahi lagta.
position: sticky, fixed se kaise different hai?
Sticky element normal element ki tarah behave karta hai jab tak scrolling mein wo specified sticking point tak nahi pohanchta — us point ke baad hi ye fixed element ki tarah chipak (stick) jaata hai. Fixed hamesha se hi fixed hota hai.
position: absolute wala element kis cheez ke relative position hota hai?
Apne nearest ancestor ke relative jiska koi position value specified ho (static ke alawa) — agar koi bhi ancestor positioned na ho, to body ke relative position hota hai.
Parent-child absolute positioning setup mein parent par kaunsi position value best rehti hai aur kyun?
position: relative — kyunke iska parent par khud koi visual impact nahi padta (jab tak top/left/right/bottom explicitly na diye jaayen), sirf ye child ke liye reference point ban jaata hai.
Agar parent par position: relative na lagayi jaaye aur child par position: absolute ho, to child kahan position hoga?
Body ke relative position hoga, jo aksar unwanted/unexpected layout result deta hai.
Kaunse do position values naya stacking context create karte hain jinki wajah se element baaki content ko cover kar sakta hai?
fixed aur absolute (aur sticky bhi jab wo stick ho jaaye).
z-index ki negative aur higher values ka kya effect hota hai?
Negative value element ko baaki elements ke peeche bhej deti hai. Higher (positive) value element ko baaki elements ke upar la deti hai.
Navigation bar, sidebar, overlay/popup jaise elements ko kis tarah ka z-index dena chahiye aur kyun?
Higher z-index — taake ye website ke baaki content ke upar hamesha visible rahein aur kisi aur element se cover na hon.
